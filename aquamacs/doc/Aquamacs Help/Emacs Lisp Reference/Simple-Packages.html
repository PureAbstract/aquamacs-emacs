40.2 Simple Packages

The simplest package consists of a single Emacs Lisp source file. In this case, all the attributes of the package (see Packaging Basics) are taken from this file.

The package system expects this .el file to conform to the Emacs Lisp library header conventions. See Library Headers.

The name of the package is the same as the base name of the .el file, as written in the first comment line. For example, given the header line: 

     ;;; superfrobnicator.el --- frobnicate and bifurcate flanges

the package name will be ‘superfrobnicator’.

The short description of the package is also taken from the first line of the file.

If the file has a “Commentary” header, then it is used as the long description.

The version of the package comes either from the “Package-Version” header, if it exists, or from the “Version” header. A package is required to have a version number. Each release of a package must be accompanied by an increase in the version number.

If the file has a “Package-Requires” header, then that is used as the package dependencies. Otherwise, the package is assumed not to have any dependencies.

A single-file package cannot have an Info manual.

The file will be scanned for autoload cookies at install time. See Autoload.
